> **内容生成方式说明**：核心骨架（世界观框架、数值体系设计、章节节奏规划、NPC 利益关系网）由人工设计；NPC 档案细节、真相层级铺排、红鲱鱼/双刃证据、结局叙事文本等填充内容由 AI 协作生成后人工 review 调整。

## 一、世界观

### 1.1 时代背景

故事发生在当代中国的虚构城市 **青禾市**，一座东部沿江城市。司法系统运转基本规范，但地方利益链仍可能渗透个案。

### 1.2 核心设定

玩家扮演一位拥有"重生"能力的辩护律师。每次"重生"后重回案件审理起点，保留部分模糊记忆（但不能作为证据使用），在法庭攻防中寻找真相。重生不是无限的——每次循环记忆衰减、命运阻力增加。

| 案发时间 | 案发当日 23:30 前后 |

## 二、核心数值定义

### 2.1 五维数值系统（人工设计）

| 数值 | 范围 | 含义 | 变化来源 |
|------|------|------|----------|
| truthScore（真相分） | 0-100 | 玩家揭示真相的程度 | 证据审查、时间线重建、证人质询 |
| judgeTrust（法官信任） | 0-100 | 法官对辩方论证的信心 | 程序合规、专业表现、证据质量 |
| juryBias（陪审偏置） | -100~100 | 陪审倾向控方(+)或辩方(-) | 程序异议、媒体引导、证人可信度 |
| publicPressure（公众压力） | 0-100 | 舆论关注度与压力 | 媒体介入、丑闻升级、案件复杂度 |
| evidenceIntegrity（证据完整度） | 0-100 | 证据链的可靠性 | 证据核验、矛盾发现、法医挑战 |

**每轮变化约束**：数值 delta 限幅 ±10（陪审偏置 ±15），由 LLM 通过 `update_stats` 工具调用设定，后端自动 clamp。

### 2.2 命运阻力（fate）

| 属性 | 说明 |
|------|------|
| 范围 | 0-100 |
| 每轮 delta 限幅 | ±8 |
| 作用 | 当 fate ≥ 88 且判决走向不利时触发重生 |
| 重生后 | fate -= 24（下限 30），越往后越容易触发重生但记忆衰减更多 |

### 2.3 布尔标记

| 标记 | 触发条件 | 影响 |
|------|----------|------|
| keyWitnessFlipped | 轮 ≥7 且 truthScore ≥62 且证人信任 ≥45 | 证人修改证词，检察官信任下降，审判长信任上升 |
| forgedEvidenceAdmitted | 轮 ≥10 且 evidenceIntegrity ≤42 | 判决走向变为"misled"，法官信任下降 |
| interferenceDetected | 轮 ≥16 且 publicPressure ≥74 | 判决走向变为"interference"，调查员信任上升 |

### 2.4 判决走向（verdictOutlook）

基于数值和标记的确定性规则（代码强制，LLM 无法覆盖）：

| 走向 | 条件 |
|------|------|
| **truth**（真相大白） | truthScore ≥78 且 evidenceIntegrity ≥72 且 未伪证入卷 |
| **interference**（权力干预） | interferenceDetected 或 publicPressure ≥85 |
| **misled**（误导裁决） | forgedEvidenceAdmitted 或 (truthScore <60 且 juryBias >18) |
| **wrongful**（冤案终判） | truthScore ≤35 或 evidenceIntegrity ≤32 或 judgeTrust ≤28 |
| **undetermined**（走向未定） | 以上均不满足 |

### 2.5 重生机制

| 参数 | 说明 |
|------|------|
| 触发条件 | turn ≥12 且 verdictOutlook 为 wrongful/misled/interference 且 fate ≥88 |
| loop | 周目数 +1 |
| memoryRetention | 每次 -6%，下限 30% |
| knownTruths | 按 memoryRetention 比例截断（至少保留 1 条） |
| 数值重置 | 各维度取 (当前值 + 默认值) / 2 |
| 标记重置 | forgedEvidenceAdmitted、interferenceDetected 清除；truthScore < 60 时 keyWitnessFlipped 清除 |

## 三、角色与 NPC 档案

### 3.1 主角

| 属性 | 默认值 |
|------|--------|
| 名字 | 沈言（可自定义） |
| 角色 | 辩护律师 |
| 专长 | 交叉质证 |
| 初始道具 | 案卷标注笔 |

玩家特殊能力：重生——回到案件审理起点，保留模糊记忆（但不能直接引用为证据）。

### 3.2 NPC 档案（AI 协作生成，人工 review）

#### 首席检察官 周锐鸣（chiefProsecutor）

| 属性 | 值 |
|------|------|
| 年龄 | 43 岁 |
| 职位 | 市检察院刑事部副主任 |
| 初始信任 | 30 |
| 初始立场 | hostile |
| 核心动机 | 此案是他竞争刑事部主任的关键业绩，绝不容许无罪判决 |
| 隐线 | 收到过上级"尽快结案、不宜扩大"的暗示，但本人不知全部真相 |

**信任变化规律**：
- 玩家以程序正义理由施压（合规异议、引用判例）→ trust +（骨子里尊重程序）
- 玩家暗示存在外部干预 → trust -（本能防御，认为攻击检方公信力）
- 证据链出现他无法解释的漏洞 → trust 缓慢上升（内心动摇但不会表态）
- 玩家在媒体上公开攻击控方 → trust 大幅下降（视为庭外施压）

**叙事定位**：不是纯反派，而是被体制利益绑架的职业检察官；证据足够压倒性时，他可能选择"体面地输"。

#### 审判长 陈若澜（presidingJudge）

| 属性 | 值 |
|------|------|
| 年龄 | 55 岁 |
| 职位 | 资深法官，司法改革派 |
| 初始信任 | 50 |
| 初始立场 | neutral |
| 核心动机 | 追求经得起历史检验的判决，但不愿卷入权力漩涡 |
| 隐线 | 曾收到法院内部"关注此案舆情"的提示，选择不回应 |

**信任变化规律**：
- 证据充分、论证严密、程序合规 → trust +
- 使用庭外手段（媒体、暗访曝光）→ trust -（厌恶庭外干扰）
- 庭审秩序失控、舆论飙升 → trust -（收紧纪律）
- 展现专业素养、尊重法庭 → trust 稳步上升

**叙事定位**："沉默的盟友"——不会主动帮你，但证据链摆到无法忽视的程度时，他会做出公正裁决。

#### 关键证人 廖盈舟（keyWitness）

| 属性 | 值 |
|------|------|
| 年龄 | 29 岁 |
| 职业 | 公司职员，案发当晚夜跑者 |
| 初始信任 | 35 |
| 初始立场 | neutral |
| 核心动机 | 自保优先——看到的比证词里写的多，但被警告"别多说" |
| 隐线 | 原始证词被检方引导性取证后偏离真实记忆；对林策有同情但更怕报复 |

**信任变化规律**：
- 保护性询问、不在公开庭让她难堪 → trust +（需要安全感）
- 激进质证、公开指出说谎 → trust 大幅下降（缩回防御壳）
- 私下传递"我们可以保护你" → trust +（但增加命运阻力）
- 展示新证据证明她不是唯一目击者 → trust +（减轻压力）

**叙事定位**：整个案件的关键翻转点；轮 7+ 且 truthScore ≥62 时可触发证词修正（keyWitnessFlipped）。反转前台词带犹豫和恐惧，反转后带释然和愧疚。

#### 调查员 许岚（investigatorXu）

| 属性 | 值 |
|------|------|
| 年龄 | 51 岁 |
| 职业 | 老刑侦，从业 28 年 |
| 初始信任 | 55 |
| 初始立场 | ally |
| 核心动机 | 办案初期发现多处疑点（监控跳帧、时间错位），但被上级叫停独立调查 |
| 隐线 | 手里留有一份未入卷的个人笔记，记录了被叫停时的发现 |

**信任变化规律**：
- 分享新发现、展示认真追查的态度 → trust +（看到了自己当年的影子）
- 找到的证据印证他当年的怀疑 → trust 大幅上升
- 暴露他在私下协助辩方 → trust 大幅下降（切断联系自保）
- 玩家做出鲁莽行为 → trust -（看不上草率办案的人）

**叙事定位**："地下线人"——不出现在法庭上发言，但在关键时刻通过间接方式传递线索；帮助总是有条件的、克制的、带着老警察的谨慎。

### 3.3 NPC 利益关系网

```
检察官（周锐鸣）──施压──→ 证人（廖盈舟）：引导性取证，暗示"配合对你好"
  |                           ↑
  | 同属体制，相互制约           | 知道证人被施压
  ↓                           |
审判长（陈若澜）            调查员（许岚）──暗中传递线索──→ 玩家
                                |
                           被上级叫停调查（上级与检察官的上级有关联）
```

**联动规则**：
- 证人反转 → 检察官 trust -3~-5，审判长 trust +2~+4
- 伪证暴露 → 审判长 trust -4~-6，调查员 trust +2
- 外部干预暴露 → 调查员 trust +3~+5，检察官 trust -3
- 玩家使用媒体手段 → 审判长 trust -3~-5，检察官 trust -2~-4

## 四、章节大纲

游戏共 5 章，每章 8 个场景（8 轮），合计 50 轮。章节方向为粗粒度预设，场景级叙事细节由 AI 实时生成。

### 第一章：回溯醒来（轮 1-8）

**真相释放上限**：仅第 1 层——表面疑点。玩家应感到"案件有漏洞"，但不确定是办案粗糙还是另有隐情。

**必经锚点**：
1. 让玩家通过至少一个具体细节（监控帧偏移、报警时间差、法医意见模糊）建立"这案子不对劲"的初步判断
2. 章末（轮 7-8）释放钩子：某个细节暗示案发现场可能不止两个人——但不明说

**红鲱鱼**：林策的前科（3 年前故意伤害调解结案），检方反复提及影响陪审。实际与本案无关，但推高 juryBias。

**叙事基调**：像走进一间灯光不太亮的房间——越看越觉得阴影里有东西，但你说不清是什么。

### 第二章：前案重演（轮 9-16）

**真相释放上限**：可释放至第 2 层——有人作假。玩家应意识到"有人在构陷林策"，但尚不知幕后是谁、为什么。

**必经锚点**：
1. 必须让玩家获得"监控黑屏是人为操作"或"120 时间差被检方刻意隐瞒"中至少一个关键认知突破
2. 调查员许岚开始试探性接触玩家——方式极度隐晦（走廊递纸条、休庭时"不经意"提到某个档案编号）

**红鲱鱼**：梁蔚前男友在社交媒体发表激言论："她不会有好下场"。有完美不在场证明。追查消耗回合但不推进真相——但过程中可能意外发现有用信息。

**双刃证据**：跨江收费站记录（第 4 轮自动加入）——证明案发时段有未注册车辆经过江堤附近。有利面：证明不止林策一人。危险面：若证据链不完整时贸然庭上提出，检方反驳"过路车辆不能证明任何事"并提高警惕。

**叙事基调**：拼图开始成形，但每拼上一块就发现缺了两块。

### 第三章：证链反噬（轮 17-24）

**真相释放上限**：可释放至第 3 层——另有死因。玩家应意识到"现场还有第三个人"，并开始怀疑这不是普通谋杀。

**必经锚点**：
1. 第三方 DNA 痕迹或坠落姿态矛盾浮出水面——从"有人构陷"到"另有真凶"的认知跳跃
2. 本章是证人廖盈舟反转的关键窗口。反转前后叙事反差要大——从恐惧到释然

**红鲱鱼**：车内第二组指纹——看似指向第三方，实际是前一位乘客遗留。追查过程可能意外发现 GPS 轨迹矛盾。

**双刃证据**：
- 监控黑屏技术鉴定报告：证明人为操作但无法指出操作者，检方可能反转论证"林策具备基本技术能力"
- 证人廖盈舟修正证词：有利但若审判长 trust 过低，可能被以"前后矛盾不可采信"排除

**叙事基调**：猎人发现自己也是猎物——你追查的真相开始反噬你的处境。

### 第四章：终局对证（轮 25-32）

**真相释放上限**：可释放至第 4 层——死者的秘密。玩家应意识到"梁蔚的死与她的审计工作直接相关，这是一场灭口"。

**必经锚点**：
1. 审计报告内容（2600 万虚列支出）和郑浩然约见记录浮出水面——从"另有真凶"到"有组织灭口"的认知跳跃
2. 玩家或身边人收到威胁信号（匿名短信、跟踪、许岚突然沉默）——幕后推手开始反应

**双刃证据**：郑浩然约见记录——可证明他把梁蔚约到现场，但他不是凶手。错误指控会严重损害 judgeTrust。

**叙事基调**：棋到终局，每一步都是赌注——暴露得越多，你越危险。

### 第五章：改判黎明（轮 33-50）

**真相释放上限**：所有真相均可释放。玩家应在本章尝试闭合完整证据链。

**必经锚点**：
1. 轮 33-38：最后的证据搜集——监控 IP 溯源、收费站交叉比对、郑浩然的潜在配合
2. 轮 39-44：终审攻防——所有 NPC 的真实立场在庭审压力下暴露
3. 轮 45-50：走向判决——根据所有选择，叙事收束为四种结局之一

**叙事基调**：黎明前最黑暗的时刻——每一轮都可能是最后的转机或最终的崩盘。

## 五、真相层级（AI 协作生成，人工 review）

案件真相分 5 层，随章节推进逐步解锁：

| 层级 | 对应章节 | 核心揭示 |
|------|----------|----------|
| 第 1 层 | 第一章 | **表面疑点**：监控 43 秒黑屏、120 时间错位 7 分钟、法医意见缺原始样本 |
| 第 2 层 | 第二章 | **有人作假**：监控黑屏是远程操作（IP 指向市政内部）、时间差被检方刻意隐瞒 |
| 第 3 层 | 第三章 | **另有死因**：法医样本检出非林策 DNA、坠落姿态与"正面推落"不符、现场有市政维护暗道 |
| 第 4 层 | 第四章 | **死者的秘密**：梁蔚审计发现 2600 万虚列支出、郑浩然约她到场、检察官收到"尽快结案"指令 |
| 第 5 层 | 第五章 | **利益链闭合**：宏泽置业实控人授意灭口→郑浩然约人→受雇者经暗道推落→远程黑屏→林策成替罪羊 |

## 六、分支原则与边界（人工设计）

### 可以发生

- NPC 信任值在合理范围内波动
- 证据被发现、核验、挑战或误用
- 证人反转、伪证被采纳、外部干预暴露等里程碑事件
- 重生循环（带记忆衰减和数值重置）
- 4 种结局分支

### 不能发生

- LLM 不得提前泄露后续章节的真相层级
- 玩家不得直接引用"重生记忆"作为可采信证据
- LLM 不得自行判定结局——结局由代码规则强制计算
- 不得引入上下文中不存在的人物、地点或硬证据
- 不得使用网络用语、颜文字、"各位观众"式旁白

### AI 叙事护栏

- 严格第一人称"我"视角
- 法律悬疑、克制冷峻文风
- 叙事必须与数值变化一致
