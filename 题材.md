# 重生 + 法律悬疑 AI互动小说游戏设计方案

---

## 一、项目定位

本项目为一个基于 Agentic AI 的互动叙事系统，玩家通过选择或自然语言输入推动剧情发展。

核心特点：

* 第一人称沉浸叙事（"我"视角）
* 重生机制（多轮尝试 + 信息不完全保留）
* 法律悬疑推理（证据 + 证人 + 判决）
* 状态驱动剧情（非纯生成）

---

## 二、核心玩法循环

```
剧情生成（第一人称）
↓
提供选项 / 接收自然语言
↓
Rule Engine解析意图
↓
更新状态
↓
进入下一轮剧情
```

---

## 三、第一人称叙事设计

### 叙事要求

* 全程使用“我”视角
* 禁止使用“玩家”“你选择了”等表达

### 示例

```
我站在法庭中央，手心微微出汗。

这个案件，我经历过一次。

那一次，我判错了。

而这一次，我不能再犯错。
```

---

## 四、重生系统设计

### 状态结构

```json
{
  "loop": 2,
  "memory_retention": 0.6,
  "known_truths": ["嫌疑人A说谎"],
  "fate": 80
}
```

### 核心机制

1. **部分记忆保留**

   * 每次重生只保留关键线索

2. **命运阻力（fate）**

   * 改变越多 → 阻力越大

3. **重生触发条件**

   * 判决失败
   * 关键节点失败
