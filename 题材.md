







# 核心玩法：
- 法律悬疑 = 推理 + 选择 + 证据
- 你的游戏本质是：玩家扮演律师 / 调查者，通过选择推进案件，并影响判决结果

每一轮：
## AI生成案件推进
- 给你选择：
- 采信证据
- 质疑证人
- 提出动机
- 判决倾向

## 你的选择影响：
- 案件走向
- NPC态度
- 最终判决


## 多结局
- 冤案
- 真相大白
- 被误导
- 权力干预


## 一个典型流程
- 案件发生 → 收集证据 → 审问证人 → 法庭辩论 → 判决结果

### 玩家每一步都要做决策：
- 是否相信证人？
- 是否提交某个证据？
- 是否反驳对方？


















五、系统架构（你必须这么做）

1️⃣ 主叙事 Agent（Story Agent）

负责：

输出剧情
给选项

输入：

当前状态
历史摘要

输出：

narrative
choices
2️⃣ 状态管理模块（State Manager）

👉 这是核心！！！

{
  "player": {
    "role": "judge"
  },
  "case": {
    "suspects": [],
    "evidence": [],
    "truth_progress": 0
  },
  "npc": {
    "lawyer_trust": 60,
    "public_opinion": 40
  }
}

👉 不要让AI自己记这些！！

3️⃣ 上下文管理（关键难点）

你必须做：

❗ 分层记忆：
🧠 短期记忆（最近3轮）

直接放prompt

📦 中期记忆（摘要）

每5轮总结一次：

“玩家已质疑证人A，发现矛盾”

🗂️ 长期记忆（结构化）

存在state里

4️⃣ 日志系统（必须做）

每一轮记录：

{
  "round": 12,
  "choice": "质疑证人",
  "summary": "发现证词矛盾",
  "state_change": {
    "truth_progress": +10
  },
  "tokens": {
    "input": 1200,
    "output": 300
  }
}
5️⃣ Token统计（很多人会翻车）

你要记录：

input tokens
cached tokens
output tokens

👉 建议：
直接用 SDK 返回字段
